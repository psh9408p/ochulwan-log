# 비용 정책

이 프로젝트는 개인 개발 및 테스트 단계에서 과금 요소가 발생하지 않는 구성을 우선합니다.

## 원칙

- 신용카드 등록이 필요한 서비스는 사용하지 않습니다.
- 무료 플랜을 초과할 수 있는 기능은 사용하지 않습니다.
- 유료 플랜, 사용량 기반 과금, 유료 애드온, 유료 API는 도입하지 않습니다.
- 과금 가능성이 있는 외부 서비스를 추가하기 전에는 반드시 사용자 승인을 먼저 받습니다.
- 로컬 개발 환경에서 실행 가능한 대안을 우선합니다.

## 현재 무료로 사용하는 구성

- Next.js 로컬 개발 서버
- 로컬 PostgreSQL
- Prisma ORM
- GitHub 기본 저장소 기능
- Telegram Bot API 기본 메시지/Webhook 기능
- Cloudflare TryCloudflare quick tunnel

## 금지 항목

- Telegram Paid Broadcasts
- Telegram Stars를 소모하는 기능
- 유료 배포 서버 또는 유료 데이터베이스
- 카드 등록이 필요한 SaaS 연동
- 무료 사용량을 초과할 수 있는 자동화 작업

## 배포 전 확인

배포 또는 외부 서비스 연결 전에는 다음을 확인합니다.

- 카드 등록이 필요한가?
- 무료 플랜 한도가 있는가?
- 한도 초과 시 자동 과금되는가?
- 유료 기능을 코드에서 호출하는가?
- 로컬 또는 무료 대안이 있는가?

위 항목 중 하나라도 비용 발생 가능성이 있으면 진행하지 않습니다.
